## 一、系统概述

### 1.1 目标

构建一个基于多Agent协作的翡翠原石智能评估系统：

- **现场评估**：通过手机拍照（含开窗打灯）+ 3D密度估算，实现快速评估，辅助购买决策
- **目标准确率**：有开窗+密度估算时达到80%+
- **核心定位**：从"预测工具"逐步演进为"决策辅助系统"，Phase 1 定位为"相似案例检索+参考意见"而非"买卖建议"

### 1.2 关键输入信息

```
现场可获取的信息（全部合法合规，无需特殊设备）：
─────────────────────────────────────────────────
✅ 多角度自然光照片（6-8张）
✅ 强光手电打灯照片（4-6张）
✅ 开窗处照片/打灯照片（最高信息量）
✅ 微距皮壳纹理照片（2-3张，含酸蚀纹检测用途）
✅ 旋转视频（10-15秒）
✅ 开窗处打灯视频
✅ 石头重量（现场称重或询问卖家）
✅ 卖家报价
✅ 声明场口
✅ 同批次其他石头的出售价格参考
✅ 环绕视频（10-15秒，用于3D重建估算密度）
✅ 参照物放置拍摄（硬币/卡片等已知尺寸物体，用于标定比例）

设备需求：手机 + 强光手电（赌石买家标配）
```

### 1.3 信息源价值分级

```
信息量对比（基于领域经验预估，待数据验证）：
─────────────────────────────────────────
纯皮壳外观         → 间接推断 → 预测天花板 60-65%
皮壳 + 打灯         → 间接+半直接 → 预测天花板 65-70%
皮壳 + 开窗         → 直接+间接 → 预测天花板 75-80%
皮壳 + 开窗 + 密度  → 多维度交叉验证 → 预测天花板 82-87%
皮壳 + 开窗 + 密度 + 切片 → 接近明料 → 预测天花板 90%+

结论：
- 开窗是现场提升准确率最关键的信息源
- 密度估算提供物理属性交叉验证，关键是检测处理货而非单纯提升品质预测
- 多信息源"交叉验证"比单一高权重信息源更可靠（减少单一模型误判的影响）
```

## 三、各Agent详细设计

### 3.1 Orchestrator Agent（编排控制器）★重点展开

#### 3.1.1 输入完整度评估

```
评估维度（5维打分，每维0-1分）：
─────────────────────────────────────────
├── 皮壳信息完整度：
│     有6张自然光 → 0.6, +4张打灯 → 0.3, +微距 → 0.1
├── 开窗信息完整度：
│     无开窗 → 0, 1个小开窗 → 0.4, 多开窗/大开窗 → 0.8-1.0
├── 密度信息可用度：
│     无重量/无环绕视频 → 0, 有环绕但重建质量差 → 0.3, 重建良好 → 0.8-1.0
├── 处理货检测覆盖度：
│     无微距 → 0.2, 有微距 → 0.6, 微距+密度 → 1.0
└── 市场信息完整度：
│     无场口/价格 → 0, 有场口 → 0.5, 场口+报价+批次 → 1.0

综合完整度 = Σ(维度得分 × 维度权重)
阈值：
  完整度 ≥ 0.7 → 全流程评估
  完整度 0.4-0.7 → 部分评估（标注信息缺口）
  完整度 < 0.4 → 提示用户补充信息，仅输出案例检索
```

#### 3.1.2 多级拒判决策

```
拒判分级（不是简单的 yes/no）：
─────────────────────────────────────────

Level 0: 不拒判（置信度 ≥ 70%）
  → 正常输出所有评估结果

Level 1: 弱拒判（置信度 55-70%）
  → 输出评估结果 + 标注"置信度偏低，建议结合其他信息判断"
  → 列出不确定性最大的1-2个因素
  → 仍然给出建议出价（但扩大安全边际至40%）

Level 2: 强拒判（置信度 40-55%）
  → 仅输出案例检索结果和相似案例统计
  → 不输出品质预测和价格建议
  → 提示具体缺失/矛盾的信息

Level 3: 完全拒判（置信度 < 40%）
  → 仅说明原因（信息不足/模型不确定/矛盾信号过多）
  → 建议：补充信息 / 找行家陪同 / 放弃

额外拒判触发条件（无论置信度如何）：
├── 矛盾信号等级 = "严重" → 至少 Level 2 拒判
├── 3D重建失败 + 无开窗 → 至少 Level 2 拒判
├── 检测到明显作假痕迹（覆膜/拼合）→ Level 3 拒判
└── 输入图片质量过低（模糊/暗光）→ Level 2 拒判，提示重新拍摄
```

#### 3.1.3 矛盾信号检测

```
矛盾检测矩阵（两两Agent输出交叉验证）：
─────────────────────────────────────────

皮壳 vs 开窗：
  皮壳好 + 开窗差 → ⚠️ 可能骗窗（卖家在最差位置开窗罕见）
  皮壳差 + 开窗好 → ⚠️ 极可能是骗窗（小开窗选最好位置）
  皮壳好 + 开窗好 → ✅ 一致，置信度提升
  皮壳差 + 开窗差 → ✅ 一致，置信度提升

开窗 vs 密度：
  开窗品质高 + 密度正常 → ✅ 一致
  开窗品质高 + 密度偏低 → ⚠️ 严重矛盾！（可能B货翻新后开窗）
  开窗品质低 + 密度正常 → 一致（种水差但为天然）
  开窗品质低 + 密度偏低 → 可能为低档处理货

密度 vs 处理货检测：
  密度正常 + 视觉检测无异常 → ✅ 高置信度A货
  密度正常 + 视觉检测有异常 → ⚠️ 可能C货（染色不改变密度）
  密度偏低 + 视觉检测有异常 → ⚠️ 极可能B货
  密度偏低 + 视觉检测无异常 → ⚠️ 需要进一步验证

皮壳 vs 声明的场口：
  皮壳特征匹配声明场口 → ✅ 一致
  皮壳特征不匹配 → ⚠️ 场口可能虚报（影响价格评估）

矛盾等级：
  无矛盾 → 正常输出
  轻度矛盾（1个维度不一致）→ 标注警告，置信度下调0.1
  中度矛盾（2个维度不一致）→ 置信度下调0.2，触发Level 1拒判
  严重矛盾（3+个维度不一致或密度vs开窗矛盾）→ 触发Level 2+拒判
```

#### 3.1.4 动态权重集成

```
品质预测Agent的集成权重不再固定，由Orchestrator动态调整：

基础权重 × 输入质量因子 × 信号一致性因子 = 实际权重

示例：
  开窗直接分类：基础权重 2.5
    输入质量因子 = f(开窗大小, 开窗数量, 图像清晰度)
    信号一致性因子 = f(与皮壳/密度是否一致)
    
  如果开窗仅1cm小窗口 + 与皮壳矛盾 → 实际权重 = 2.5 × 0.5 × 0.6 = 0.75
  如果开窗5cm + 与皮壳一致 → 实际权重 = 2.5 × 1.0 × 1.0 = 2.5

动态调整规则：
├── 输入质量高 + 信号一致 → 权重放大 1.0-1.2×
├── 输入质量低 or 信号矛盾 → 权重衰减 0.4-0.8×
└── 某Agent输出明显离群 → 该Agent权重临时降为0（排除该模型本轮投票）
```

  声明场口 ∉ TOP3推断场口 → ⚠️ 场口可能虚报

```

### 3.4 处理货检测Agent ★全新

#### 3.4.1 为什么需要独立的处理货检测Agent

v3.2将A/B货鉴别几乎完全依赖3D密度估算，但存在致命盲区：

```
密度法的局限性：
─────────────────────────────────────────
❌ C货（纯染色）：密度完全正常，3.30-3.35 g/cm³
❌ 部分B+C货：密度可能3.20-3.30，落入正常翡翠范围边界
❌ 覆膜处理：密度正常
❌ 拼合石：局部密度不正常但整体平均可能正常
❌ 密度估算误差 ±0.15 g/cm³ 可能导致误判

结论：密度是必要条件，不是充分条件。
      需要视觉检测作为独立的信息源交叉验证。
```

#### 3.4.2 处理货分类体系

```
天然A货：未经任何化学处理，仅物理切割/打磨/抛光
─────────────────────────────────────────

B货（酸洗注胶）：
  原理：用强酸浸泡去除翡翠中的杂质/铁质，然后注入树脂/胶
  目的：改善透明度、颜色，让低档料看起来高档
  视觉特征：
    ├── 酸蚀纹（"龟裂纹"/"蜘蛛网状"）：强酸腐蚀后在表面留下的网状裂纹
    ├── 树脂光泽（"塑料感"）：注胶后光泽异常，缺乏天然翡翠的玻璃光泽
    ├── 颜色漂浮感：颜色分布不自然，像"浮在表面"
    ├── 基底变白/变浊：酸洗后结构空洞导致
    └── 荧光反应（紫外灯下）：注入的树脂通常有荧光（但现场紫外灯不方便）

C货（染色处理）：
  原理：将有色染料渗入翡翠内部或裂隙中
  目的：赋予翡翠颜色（绿色/紫色/红色等）
  视觉特征：
    ├── 颜色沿裂隙富集（"色筋"）：染料沿裂纹集中分布
    ├── 颜色不自然（过于鲜艳/均匀）：天然翡翠颜色通常有层次过渡
    ├── 颜色褪色/变色风险（长期）：染料不稳定
    └── 无色根（缺乏天然色的"根"部结构）

B+C货（酸洗+注胶+染色）：
  同时具有B货和C货的特征

覆膜处理：
  原理：在翡翠表面覆一层有色或增透的膜
  视觉特征：
    ├── 表面光泽异常
    ├── 边缘膜层可见（放大观察）
    └── 刮擦测试（现场不做）

拼合石：
  原理：多块翡翠/仿品粘合在一起，通常是上层翡翠+下层仿品
  视觉特征：
    ├── 侧面可见拼合缝
    ├── 不同角度颜色/光泽有差异
    └── 打灯透光不均匀（不同材料透光不同）

合成翡翠（人造）：
  原理：高温高压合成或玻璃/塑料仿制
  视觉特征：
    ├── 过于完美（无任何天然特征）
    ├── 内部过于干净
    └── 密度/折射率异常
```

#### 3.4.3 检测模型设计

```
处理货检测由3个子模型组成：

子模型1：酸蚀纹检测（B货专用）
─────────────────────────────────────────
  模型：ResNet50 with Attention（细粒度纹理分类）
  输入：微距皮壳/开窗照片（2-3张，含多角度）
  输出：酸蚀纹概率 [0,1] + 热力图（异常区域高亮）
  训练数据：
    - B货翡翠微距照片 2000+（含不同酸洗程度的样本）
    - A货翡翠微距照片 3000+（正常纹理对照）
    - 关键挑战：天然石纹 vs 酸蚀纹的区分（需专家标注）
  
  注意：开窗处如果有酸蚀纹最容易检测
       全皮壳原石的B货检测难度最大（皮壳遮住了处理痕迹）

子模型2：染色检测（C货专用）
─────────────────────────────────────────
  模型：EfficientNet-V2-M + 颜色空间分析
  输入：开窗照片 + 打灯照片（打灯下颜色分布更明显）
  额外特征工程：
    - 颜色沿裂隙分布分析（边缘检测 + 颜色梯度）
    - 颜色直方图分析（天然色 vs 人工色的分布差异）
    - HSV空间异常检测
  输出：染色概率 [0,1] + 可疑区域掩码
  训练数据：
    - C货翡翠照片 1500+
    - A货有色翡翠照片 2000+（关键：学习区分天然色和人工色）

子模型3：综合异常检测（覆膜/拼合/合成）
─────────────────────────────────────────
  模型：多输入异常检测（AutoEncoder + One-Class SVM）
  输入：皮壳/开窗特征向量 + 密度值 + 场口信息
  输出：综合异常分数 [0,1]
  原理：学习天然翡翠的多维特征分布，检测离群点
  
  补充规则：
    - 密度 < 2.90 → 几乎不可能是翡翠（直接判为仿品）
    - 密度 2.90-3.15 → 高风险区间（B货/仿品重叠区）
    - 密度 3.15-3.25 → 疑似B货/部分C货
    - 密度 3.25-3.40 → 正常范围
    - 密度 > 3.40 → 异常（需复查）

三个子模型的融合：
  综合A货概率 = f(酸蚀纹(-), 染色(-), 密度, 异常检测)
  
  IF 酸蚀纹概率 > 0.7 OR 密度 < 3.10:
    → 高风险B货，强烈建议放弃
  IF 染色概率 > 0.6 AND 密度正常:
    → 疑似C货，建议送检
  IF 全部低概率 + 密度正常:
    → 倾向A货，置信度根据信息完整度调整
```

#### 3.4.4 数据采集策略

```
处理货训练数据获取：
─────────────────────────────────────────
1. 检测机构合作（关键渠道）
   - 与翡翠检测实验室合作，获取已鉴定的A/B/C货照片

### 3.5 三维密度估算Agent（修订：增加可行性验证与降级策略）

#### 3.5.1 不同皮壳类型的3D重建可行性预估

```
关键认知：不是所有翡翠原石都适合3D重建

皮壳类型 vs 3D重建可行性：
─────────────────────────────────────────
白沙皮     → 纹理清晰，对比度高 → 重建成功率 80-90%（最优）
黄沙皮     → 纹理较丰富 → 重建成功率 70-85%
铁锈皮     → 纹理丰富 → 重建成功率 70-80%
水翻砂     → 皮壳粗糙，纹理好 → 重建成功率 65-80%
黑乌砂     → 深色+反光 → 重建成功率 30-50%（最差！）
石灰皮     → 颜色均匀+纹理少 → 重建成功率 40-60%
蜡皮       → 表面光滑+反光 → 重建成功率 25-45%
土豆皮     → 纹理中等 → 重建成功率 55-70%

反光问题的严重性：
  - 黑乌砂是缅甸最常见的场口之一（莫湾基/麻蒙等）
  - 如果黑乌砂重建成功率仅30-50%，影响面很大
  - 需要在拍摄指引中针对反光皮壳给出特殊指导（如用柔光/多角度）
```

#### 3.5.2 重建质量自动评估

```
在3D重建完成后，自动评估重建质量：

质量评估指标：
├── 关键帧覆盖率（目标 > 270° 环绕角度）
├── 有效特征点数量（SfM中成功匹配的特征点）
├── 点云密度（重建的稠密点数/体积比）
├── 重投影误差（相机位姿估计的精度指标）
├── 网格完整性（是否有大面积空洞）
└── 底部遮挡比例

质量分级：
  优（覆盖率>270° + 点云密度>阈值 + 无大面积空洞）
    → 体积误差估计：5-8%，密度误差：±0.10-0.18 g/cm³
  
  良（覆盖率>220° + 少量空洞可插值补全）
    → 体积误差估计：10-15%，密度误差：±0.15-0.25 g/cm³
  
  差（覆盖率<220° or 大面积空洞 or 特征匹配失败）
    → 标记为"3D重建质量不足"，密度结果不纳入A/B货判断
    → 降级方案：仅使用CNN体积回归（方案C）做粗筛

输出中必须标注重建质量等级，让用户知道密度估算的可信度。
```

#### 3.5.3 降级策略

```
当方案B（SfM+MVS）失败时的降级路径：

Level 1 降级：方案B → 方案C（CNN直接回归体积）
  触发条件：SfM特征匹配点数 < 阈值
  影响：体积误差从5-8% → 10-15%
  密度用途：仅粗筛（区分翡翠 vs 明显仿品如水沫子）

Level 2 降级：跳过密度估算
  触发条件：方案C也失败 or 无环绕视频 or 无重量
  影响：输出中标注"密度信息不可用"
  A/B货鉴别完全依赖处理货检测Agent（视觉检测）

Level 3 降级：降级到仅案例检索
  触发条件：多个Agent不可用
  仅输出相似案例，不做预测
```

  每周自动更新一次
  公盘结束后48小时内紧急更新

├── 公盘成交记录：权重1.0（最可靠，公开透明）
├── 检测机构备案：权重0.9（有鉴定背书）

### 3.8 历史对比Agent（修订：冷启动阶段的核心Agent）

#### 冷启动策略

```
在模型训练完成前，历史对比Agent是唯一可用的功能：

Phase 0.5（数据积累期）的产品形态：
  "翡翠原石案例检索工具"

  功能：
  ① 拍照 → 提取特征 → 搜索相似案例
  ② 展示最相似的 N 个案例（含品质+价格+切割结果）
  ③ 统计相似案例的涨跌比、品质分布、价格分布
  ④ 不做预测，只做信息聚合展示
  ⑤ 明确标注："以下为相似案例参考，非AI预测"

  为什么先做案例检索：
  - 不需要标注数据（只要有案例库即可）
  - 技术门槛低（向量检索+特征提取）
  - 用户仍然获得价值（省去人工搜索案例的时间）
  - 建立用户信任（先让用户习惯使用，再逐步加入AI预测）
  - 积累用户反馈数据（用户点击了哪个案例→隐式标注相似度）

  从案例检索过渡到AI预测的路径：
  Phase 0.5: 案例检索（无需标注）
  Phase 1.0: 案例检索 + 简单统计推断（"相似案例中65%为中档"）
  Phase 1.5: 案例检索 + ML预测（标注数据达到最低门槛后）
  Phase 2.0+: 端到端AI预测（数据充分后）
```

#### 案例相似度计算

```
多模态相似度 = Σ(w_i × sim_i)

相似度维度：
├── 视觉相似度（皮壳特征向量cosine距离）: 权重 0.35
├── 开窗相似度（开窗种水特征距离）: 权重 0.25
├── 结构相似度（场口+重量+形状）: 权重 0.20
├── 皮壳特征相似度（松花/蟒带/癣的Jaccard距离）: 权重 0.15
└── 上下文相似度（同批次/同卖家）: 权重 0.05

注意：不同维度在不同场景下可用性不同
  - 无开窗时，开窗相似度权重=0，其他维度权重等比放大
  - 特征缺失时相应维度退化为0
```

### 6.3 训练策略（针对数据不足的缓解措施）

```
数据不足时的技术缓解：
─────────────────────────────────────────

2. 数据增强：
   几何增强：旋转、翻转、裁剪、透视变换
   光度增强：亮度、对比度、色温、添加噪声
   打光模拟：模拟不同角度/强度的手电打光
   开窗增强：在不同位置/大小模拟开窗（CutMix变体）
   皮壳增强：模拟不同湿度/油性对皮壳外观的影响

3. 自监督预训练：
   用无标注数据（大量直播截取的未标注照片）做自监督预训练
   方法：SimCLR / MAE / DINO
   预期：减少30-50%标注数据需求

4. 半监督学习：
   少量标注数据 + 大量未标注数据
   方法：FixMatch / FlexMatch
   适用：皮壳分类、开窗种水分类（类别边界较清晰）

5. 课程学习：
   先训练容易区分的类别（玻璃种 vs 豆种）
   再训练困难边界（冰种 vs 糯冰种）
   先训练A货 vs 明显B货，再训练边界样本

6. 主动学习：
   模型训练 → 找到最不确定的样本 → 优先标注这些样本
   循环迭代，用最少的标注量达到最好的效果
```

### 8.3 第一层：零样本视觉模型详细设计

#### 8.3.1 CLIP — 零样本分类 + 特征提取

```
为什么选 CLIP 而不是 EfficientNet：
─────────────────────────────────────────
EfficientNet: ImageNet 1000 类 → 学的是"狗/猫/车"的特征
              → 对翡翠皮壳几乎没有区分能力

CLIP: 4亿图文对训练 → 学会了"任何图像对应任何文本描述"的映射
      → 可以通过文本 prompt 做任意分类，不需要训练！
      → 同一模型同时输出分类结果 + 图像 Embedding

模型选型: openai/clip-vit-large-patch14 (ViT-L/14)
  参数量: 428M (下载 ~1.7GB)
  输出: 768维 Embedding
  推理速度: ~150ms/图 (CPU), ~20ms/图 (GPU)
  
备选 (更快): laion/CLIP-ViT-B-32 (ViT-B/32)
  参数量: 151M, 512维 Embedding, ~60ms/图 (CPU)
  精度略低于 ViT-L, MVP可以先从B-32开始
```

**CLIP Zero-shot 分类方法**：

```
原理: 将分类问题转化为"图片和哪个文本描述最匹配"

Step 1: 构造候选文本 prompt
  例: ["白沙皮翡翠原石", "黄沙皮翡翠原石", "黑乌砂翡翠原石", ...]

Step 2: CLIP 分别计算图片和每个文本的匹配度 (cosine相似度)

Step 3: softmax → 得到各类别概率

代码等价逻辑 (不写代码, 仅说明原理):
  image_features = CLIP.encode_image(photo)        # (768,)
  text_features = CLIP.encode_text(prompts)         # (N, 768)
  similarity = image_features @ text_features.T     # (N,)
  probs = softmax(similarity / temperature)

多个prompt取平均 (提高稳定性):
  "白沙皮翡翠原石"
  "一块白沙皮翡翠赌石"
  "白色风化石皮的翡翠原石"
  → 三个prompt的text_features取平均 → 更鲁棒的分类
```

#### 8.3.2 SAM2 — 零样本分割

```
模型: facebook/sam2-hiera-large (Hiera-L backbone)
  参数量: 224M (~900MB)
  特点: 可以用"点/框/文本"作为 prompt 来分割任意物体
        不需要训练，开箱即用

MVP 中的分割任务:

任务1: 裂纹检测与量化
  Prompt方式: 在图片上用点标注裂纹区域 → SAM2 输出分割掩码
  自动化策略:

    ├── 裂纹位置: [左侧, 底部, 背面]
    ├── 裂纹长度: [3.2cm, 1.5cm, 0.8cm]
    ├── 裂纹走向: [斜向, 横向, 纵向]
    └── 裂纹掩码图 (用于 LLM Agent 参考和可视化)

  Prompt方式: 图片中心区域加点 → SAM2 分割开窗区域

    ├── 开窗占石头表面比例: ~15%

  Prompt方式: 对绿色区域(松花)/黑色区域(癣)/条带区域(蟒带)分别分割
  输出: 各特征的位置、面积、分布密度

注意: SAM2 的推理在 CPU 上较慢 (~2-5秒/图)，
      MVP 可以用较小的 Hiera-B 或 Tiny 变体加速
```

#### 8.3.3 Depth Anything V2 — 单目深度估计

```
模型: depth-anything/Depth-Anything-V2-Large
  参数量: 335M (~1.3GB)
  特点: 单张 RGB 图 → 相对深度图，不需要双目/结构光
        在各种材质上泛化能力极强（包括石头）

MVP 中的用途:

用途1: 形状分析
  从多角度深度图拼合 → 石头的大致 3D 形状
  输出: 长宽高比例、规整度评分

用途2: 粗略体积估算
  前提: 照片中有参照物 (硬币/卡片)，用于尺度标定
  方法: 深度图 × 参照物标定 → 像素→厘米转换 → 体积积分
  精度: 粗略 (±20-30%)，不替代专业测量，但可为 LLM 提供参考
  输出: {估算体积: "约 280-350 cm³", 置信度: "低, 仅供参考"}

用途3: 皮壳凹凸/纹理分析
  深度图反映皮壳的粗糙度/砂粒粗细
  辅助 LLM 判断种老/种嫩

注意: 深度图是一个额外的信号源，不要求高精度。
      MVP 中它的角色是"给 LLM 多一个维度的信息参考"，
      即使精度不高，LLM 也能通过常识推理来合理使用。
```

### 8.4 第二层：CLIP Embedding 案例检索

```
为什么用 CLIP Embedding 而不是 EfficientNet：
─────────────────────────────────────────
- CLIP 的 Embedding 空间是"图文对齐"的
  → "冰种翡翠" 这个文本和对应的图片在向量空间中接近
  → 可以用文本搜索图片，也可以用图片搜索图片

- 案例检索时, 可以用描述作为查询:
  "木那场口白沙皮冰种飘花翡翠原石" → 直接搜到相关案例

检索流程:
  1. 用户照片 → CLIP encode → 768维归一化 Embedding
  2. 多张照片 Embedding 取平均 → 整块石头 Embedding
  3. 与案例库 Embedding 矩阵做内积 (L2归一化后 = cosine相似度)
  4. Top-10 案例

增量加权方案 (第二阶段):
  开窗照片 CLIP Embedding 权重 ×1.5
  打灯照片 CLIP Embedding 权重 ×1.2
  自然光照片 CLIP Embedding 权重 ×1.0

案例库存储:
  cases.json: 案例元数据 (品质/价格/结果/标签)
  clip_features.npy: 预计算的 CLIP Embeddings (M × 768)
  images/: 案例缩略图
```

### 8.5 第三层：LLM Agent 推理引擎（核心）

**这是整个 MVP 最关键的部分**——LLM Agent 代替了传统方案中的所有手写规则和训练模型。

#### 8.5.1 为什么 LLM Agent 可以胜任

```
现代多模态 LLM 的能力 (Claude/GPT-4V/Qwen2.5-VL):

1. 视觉理解: 能"看懂"照片中的物体、纹理、颜色、透明度
   → 看翡翠照片 ≈ 一个初级翡翠玩家的水平，但更稳定

2. 跨图对比: 能同时看多张照片，比较不同角度的特征
   → 自然光 vs 打灯 vs 开窗，LLM 可以交叉验证

3. 常识推理: 知道"透明度高的石头一般更值钱"、"有裂缝会影响价值"
   → 这些常识是模型内化的，不需要我们教

4. 结构化输出: 可以输出严格格式的 JSON
   → 前端直接渲染，不依赖自然语言解析

5. 领域知识可注入: System Prompt 中提供翡翠专业知识
   → 相当于给 LLM "培训"了翡翠鉴定知识

LLM Agent 做不到的 (需要视觉模型补充的):
  ❌ 精确测量尺寸 (需要参照物 + 深度估计)
  ❌ 像素级分割 (需要 SAM2)
  ❌ 精确的特征向量检索 (需要 CLIP Embedding)
  → 这些由第一层和第二层补充
```

**领域知识注入示例** (Prompt 片段):

```
# 翡翠原石品质评估知识

## 皮壳类型与品质关系
- 白沙皮 (木那): 多为冰种/糯冰，飘花概率高，品质最好的皮壳之一
- 黑乌砂 (莫西沙/莫湾基): 皮壳越油亮种越老，灯下透光好则内部品质高
- 黄沙皮 (大马坎): 水石为主，品质较稳定，糯种-糯冰多见
- 水翻砂 (会卡): 大料多，种水跨度极大，从冰种到豆种都有可能
- 石灰皮 (百山桥): 多为嫩种，砖头料概率高
- 铁锈皮 (后江): 皮薄，以出高色著称，种水一般为糯种

## 种水分级标准 (从高到低)
1. 玻璃种: 完全透明，可见下方文字，强荧光 → 极品，市价极高
2. 高冰种: 接近全透明，起荧光 → 高档，手镯可达百万级
3. 冰种: 半透明，有荧光感 → 中高档，手镯¥10-50万
4. 糯冰种: 微透明，略有浑浊 → 中档，手镯¥2-10万
5. 糯种: 不透明但质地细 → 普通，手镯¥5000-2万
6. 豆种: 不透明，颗粒粗 → 低档，手镯¥500-3000
7. 砖头料: 石性重，无玉质感 → 废料，不值得加工

## A/B/C货鉴别
- A货(天然): 玻璃光泽，天然纹理，放大可见"翠性"（解理面反光）
- B货(酸洗注胶): 蜘蛛网状酸蚀纹，树脂光泽（不自然），颜色漂浮感
- C货(染色): 颜色沿裂隙集中分布，过于鲜艳/均匀，可褪色
- 关键: 密度无法检测C货！C货密度完全正常！必须依赖视觉判断

## 开窗选择性偏差
- 卖家总是在石头最好的位置开窗
- 小开窗+品质极好+皮壳表现一般 = 极可能是"骗窗"
- 多面大开窗+品质一致+皮壳好 = 较高可信度
- 整体品质通常比开窗处差半级到一级
```

#### 8.5.6 增量分析策略（优化延迟体验）

```
用户等待体验优化：不要等所有分析完成才显示结果。

流式返回方案 (SSE, Server-Sent Events):

时间线:
  0.0s  用户点击"开始分析"
  0.2s  → SSE event: {"stage": "uploading", "progress": 0.8}
  0.5s  → SSE event: {"stage": "clip_analysis", "message": "正在分析皮壳类型..."}
  1.0s  → SSE event: {"stage": "clip_done", "data": {"skin_type": "白沙皮", "zhuoshui": "冰种"}}
        → iPhone 立即显示: "皮壳类型: 白沙皮 ✓ 种水初判: 冰种 ✓"
  1.5s  → SSE event: {"stage": "sam2_done", "data": {"cracks": 1, "crack_location": "左侧"}}
        → iPhone 追加: "裂纹: 1条 (左侧)"
  2.0s  → SSE event: {"stage": "case_retrieval_done", "matches": [...]}
        → iPhone 追加: "找到 10 个相似案例"
  2.5s  → SSE event: {"stage": "llm_reasoning", "message": "LLM 综合分析中..."}
  5.0s  → SSE event: {"stage": "complete", "data": {完整 JSON}}
        → iPhone 展示完整评估报告

关键: 用户在 1 秒内就能看到第一批结果（CLIP 分类），
      而不是等 5-8 秒才看到任何东西。
```

### 8.6 后端 API 设计（修订）

```
后端项目结构:

backend/

├── models/
│   ├── clip_model.py          # CLIP 封装 (zero-shot分类 + embedding)

├── services/
│   ├── llm_agent.py           # LLM API 调用 + prompt 组装
│   ├── case_store.py          # 案例库 CRUD + CLIP embedding 检索
│   └── context_builder.py     # 组装发送给 LLM 的结构化上下文
├── data/
│   ├── domain_knowledge.txt   # 领域知识文本 (注入 System Prompt)
│   ├── clip_prompts.json      # CLIP zero-shot 分类的 prompt 模板
│   ├── cases.json             # 案例元数据
│   └── clip_features.npy      # 预计算 CLIP Embeddings
├── images/                    # 案例图片
├── uploads/                   # 用户上传 (定期清理)
└── requirements.txt

API 端点:

POST /api/evaluate          ← 核心端点 (SSE 流式返回)
  multipart/form-data:
    - images: File[]
    - weight_g: float?
    - mine: str?
    - ask_price: float?

GET /api/cases              ← 案例列表
POST /api/cases             ← 添加案例 (管理用)
GET /api/cases/{id}         ← 案例详情

技术栈:
  语言: Python 3.11

1. 独立处理货检测Agent
   - 不依赖密度单一指标，视觉+密度交叉验证
   - 覆盖B货（酸蚀纹）、C货（染色）、覆膜、拼合石
   - 这是业内首个系统化处理货视觉检测方案

2. 开窗分析 + "开窗→整体"推断模型
   - 业内首个系统性利用开窗信息做AI预测
   - 开窗是合法的、免费的、信息量极大的输入
   - 解决选择性偏差问题是核心创新

3. 多维度矛盾检测机制
   - 不简单加权平均，而是交叉验证各Agent输出
   - 矛盾本身就是强信号（可能骗窗/作假/异常品）
   - 矛盾检测可以比单一模型更早发现异常

4. 冷启动策略：从案例检索到AI预测的渐进路径
   - Phase 0.5即可上线提供价值（案例检索）
   - 不要求标注数据就能运行
   - 建立用户信任和技术验证同步进行

5. 端云混合推理
   - 离线可用（缅甸矿区无网络也能用）
   - 隐私保护（照片可选不上传）
   - 成本优化（端侧处理大部分请求）

6. 数据飞轮 + 主动学习
   - 用户切割回传 → 模型自动改进
   - 主动学习优先标注难样本
   - 随着时间推移，系统越来越好
```

## 附录D：术语表

| 术语 | 解释 |
|------|------|
| 种水 | 翡翠的透明度和质地，从高到低：玻璃种>冰种>糯种>豆种 |
| 开窗 | 在原石皮壳上磨开一个小口，露出内部玉肉 |
| A货 | 天然翡翠，未经化学处理 |
| B货 | 酸洗注胶处理的翡翠 |
| C货 | 染色处理的翡翠 |
| 松花 | 皮壳上的绿色表现，指示内部可能有色 |
| 蟒带 | 皮壳上的条带状凸起，指示内部种水走向 |
| 癣 | 皮壳上的黑色斑点/斑块，内部可能有黑 |
| 场口 | 翡翠原石的产地/矿区 |
| 皮壳 | 翡翠原石表面的风化层 |
| 水沫子 | 一种常见的翡翠仿品（钠长石玉），密度约2.60 |
| 拒判 | 当模型不确定时拒绝给出预测，避免错误引导 |
