# 翡翠原石AI评估 — 第一阶段 MVP 实施计划

> 版本：MVP v4.0 | 日期：2026-05-25
> 架构：国产最强模型栈 — 通义千问 VL Max + DeepSeek V4 Pro
> 核心思路：**皮壳→内部 概率性推断**，每项推断标注依据、置信度、反例、案例支持
> MVP 约束：**暂不考虑开窗**，仅基于皮壳+打灯做推断

---

## 〇、为什么这个方向是对的

### 赌石的本质

赌石不是"瞎猜"，是**贝叶斯推断**。

一块石头的皮壳和内部品质之间存在明确的因果链条：

```
皮壳类型 → 场口特征 → 该场口的典型产出品质区间
砂粒粗细 → 晶体结构 → 种老(细密) 还是 种嫩(粗松)
松花     → 绿色分布 → 有绿/无绿/绿的走向和浓度
蟒带     → 色带走向 → 蟒下是否有色, 色的范围
癣       → 杂质或伴生绿 → 癣吃色 还是 绿随癣走
打灯透光 → 种水好坏 → 透得深=种老, 不透=种嫩
打灯颜色 → 底色色调 → 偏绿/偏白/偏黄/偏紫
裂纹     → 价值折损 → 表面裂还是深入内部
```

一个有经验的赌石买家，看到一块白沙皮、砂粒细密、松花散布、打灯透光好——心里已经有数了。他不确定的是：**内部到底多好、有没有隐藏的裂、绿色能不能吃进去**。而这些，正是系统要做的概率性推断。

### v3.0 的问题：矫枉过正

v3.0 为了防止 LLM 编造，把 `internal_quality` 整段写死为"无法评估"。这等于把产品最有价值的部分扔掉了。

**买家不需要你再告诉他"皮壳是白沙皮"——他已经看到了。他需要的是：基于这些皮壳特征，内部大概是什么样、有多大的把握、最大的风险是什么。**

### v4.0 的解法

不回避推断，而是**规范推断**。每个关于内部品质的推断必须附带：

| 要素 | 说明 | 示例 |
|------|------|------|
| 观测依据 | 这个推断基于哪个皮壳特征 | "砂粒极细密 + 打灯透光>3cm" |
| 置信度 | 0-1, 明确数值 | 0.65 |
| 推理链条 | 从观察到结论的逻辑路径 | "砂细→晶体结构致密→冰种特征" |
| 反例/风险 | 什么情况下这个推断会错 | "也可能仅是表皮细腻, 内部变粗" |
| 案例支持 | 相似案例的统计数据 | "案例库 5 块类似皮壳, 3 块冰种, 2 块糯冰" |

## 一、MVP 一句话

**iPhone 拍照 → 通义千问 VL Max 识别诊断性皮壳特征 → DeepSeek V4 Pro 概率性推断内部品质 → 每项推断标注依据+置信度+反例+案例支持 → 输出评估报告**

核心场景：站在赌石摊前，手机拍一组照片，系统基于皮壳特征做概率性内部推断，明确告诉你"有 65% 把握是冰种，但这块石头的最大风险是左侧那条裂可能吃进去了——案例库里类似的情况有 40% 切垮"。

---

## 二、核心原则

| 原则 | 说明 |
|------|------|
| **概率性推断, 不回避** | 基于皮壳特征推断内部品质, 但必须是概率的, 不是确定的 |
| **每项推断可追溯** | 结论必须标注：观测依据→推理链条→置信度→反例→案例支持 |
| **不确定性量化** | 置信度用 0-1 数值, 不用"可能""大概"等模糊词 |
| **皮壳特征的诊断价值分级** | 打灯透光 > 砂粒粗细 > 松花 > 蟒带 > 皮壳颜色——权重不同 |
| **案例驱动** | "5 块类似皮壳的石头 4 块切涨" > "从理论上讲这个皮壳应该好" |
| **国产最强模型栈** | 视觉：通义千问 VL Max；推理：DeepSeek V4 Pro Think Deep |

## 三、皮壳→内部 推断体系

### 3.1 诊断性特征分级

```
┌─────────────────────────────────────────────────────────┐
│ 一级特征 (直接反映内部结构, 权重最高)                      │
│                                                          │
│ 砂粒粗细度 → 晶体结构致密度 → 种老/种嫩                   │
│   - 细密如粉: 内部大概率冰种及以上 (置信度 0.7)            │
│   - 中等偏细: 糯冰-冰种区间 (置信度 0.6)                  │
│   - 粗松: 豆种-糯种 (置信度 0.8, 粗松是最可靠的负向指标)   │
│                                                          │
│ 打灯透光深度 → 种水的核心证据                              │
│   - 全透>3cm+光圈大: 冰种及以上 (置信度 0.75)              │
│   - 中等1-2cm: 糯冰-冰种 (置信度 0.6)                     │
│   - 仅表皮下: 豆种-糯种 (置信度 0.8)                      │
│                                                          │
│ 打灯透光颜色 → 底色/飘色                                  │
│   - 偏绿: 内部可能有绿底/飘绿                             │
│   - 偏白: 无色/白色底                                     │
│   - 偏黄: 黄翡底                                          │
│   - 偏紫: 紫罗兰底                                        │
├─────────────────────────────────────────────────────────┤
│ 二级特征 (指示颜色和结构异常, 权重中等)                     │
│                                                          │
│ 松花 (位置/密度/颜色/形态) → 内部是否有绿和绿的走向        │
│   - 带状松花: 很可能有绿带 (置信度 0.7)                    │
│   - 点状稀疏: 可能有零星绿 (置信度 0.5)                    │
│   - 松花色深: 绿色较浓                                      │
│   - 松花色浅: 淡绿或无色                                    │
│   - "绿随松花走" 是行业共识，但松花不进入内部的情况也存在    │
│                                                          │
│ 蟒带 (宽窄/走向/清晰度/颜色) → 内部色带                    │
│   - 清晰宽蟒: 蟒下大概率有色带 (置信度 0.75)               │
│   - 模糊窄蟒: 可能有浅色带 (置信度 0.5)                    │
│   - 蟒带凸起: 内部结构可能致密                              │
│                                                          │
│ 癣 (颜色/位置/与松花关系)                                   │
│   - 黑癣在绿松花旁边: "癣夹绿" — 好信号 (置信度 0.6)       │
│   - 癣在松花正上方: "癣吃色" — 坏信号 (置信度 0.6)          │
│   - 癣孤立, 无松花相伴: 纯杂质, 负面影响                     │
├─────────────────────────────────────────────────────────┤
│ 三级特征 (辅助判断, 权重较低)                               │
│                                                          │
│ 皮壳类型 → 场口 → 该场口典型品质先验                       │
│   - 木那白沙皮: 冰种概率较高                               │
│   - 莫西沙黑乌沙: 冰糯种居多                               │
│   - 后江水翻砂: 糯种居多                                   │
│                                                          │
│ 皮壳颜色均匀度 → 内部结构稳定性                            │
│ 表面裂纹 → 裂纹深度的概率推断                               │
│   - 细如发丝在表面: 深入概率 30%                           │
│   - 肉眼可见已开口: 深入概率 70%                           │
│   - 多条平行裂纹: 内部结构可能疏松                          │
│                                                          │
│ 处理痕迹 → B/C 货风险                                      │
└─────────────────────────────────────────────────────────┘
```

### 3.2 多信号交叉验证

单信号不可靠，多信号交叉验证才可靠：

```
砂粒细密 (正向, +0.2)
打灯透光好 (正向, +0.25)
松花带状分布 (正向, +0.15)
左侧开口裂 (负向, -0.3)
─────────────────────
综合: 内部可能冰种飘绿, 但左侧裂风险高

如果只有"砂粒细密"一条正向信号 → 置信度低, 不建议出强结论
如果砂粒+打灯+松花+蟒带 四条正向 → 置信度高, 可以出较强结论
如果信号相互矛盾 (砂细但打灯不透) → 标注信号矛盾, 建议谨慎
```

### 5.1 为什么是这个架构

```

  ──────────────────────────────────────────────────────────────────────────
  v1.x CLIP桥接        极弱(≈随机)      强(V4 Pro)    低          ¥0.05
  v3.0 "无法评估"      强(Qwen VL)      强(V4 Pro)    中*          ¥0.10-0.25
                                                        *推理被"无法评估"限制
  
  v4.0 概率性推断 ★    强(Qwen VL)      极强(Pro      高          ¥0.12-0.30

**核心变化**：v3.0 把推理模型的手绑住了（"内部品质不许推断"）。v4.0 放手让它推断，但用**五要素框架**锁住质量——每条推断都可追溯、可质疑、可验证。

### 5.3 模型详情与 API 配置

#### 视觉模型：通义千问 VL Max

```
API 接入:
  端点: https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions
  模型: qwen-vl-max
  输入: System Prompt + 8张照片 (自然光4-6 + 打灯2-4 + 微距1-2)
  输出: 结构化诊断性特征观察报告 (~1500-3000 tokens)

优势:
  - 中文原生多模态, 对"种水""皮壳""松花""蟒带""癣"等术语天然熟悉
  - 阿里云国内节点, 延迟 <500ms
  - 支持多图输入 (最多几十张)

降级链:
  Qwen-VL-Max → Qwen-VL-Plus → Qwen2.5-VL-72B 自部署
```

#### 推理模型：DeepSeek V4 Pro Think Deep

```
API 接入:
  端点: https://api.deepseek.com/v1/chat/completions
  模型: deepseek-reasoner
  参数:
    temperature: 0.1
    max_tokens: 8192
    thinking: {"type": "enabled", "budget": 4096}

为什么 thinking budget 要 4096:
  - 需要逐条分析 5-8 个诊断性特征的诊断价值
  - 需要交叉验证: 砂粒×打灯×松花×蟒带×裂纹 → 是否存在矛盾信号
  - 需要结合案例统计做贝叶斯更新
  - 需要为每个推断生成五要素
  - 需要做综合风险评估 (多维度权衡)
  
  2048 可能不够, 4096 是安全值

单次成本: ¥0.07-0.15
```

#### 案例检索：Chinese-CLIP

```
模型: OFA-Sys/chinese-clip-vit-large-patch14 (HuggingFace)
用途: 图像相似度检索 → Top-5 历史案例
成本: 本地 CPU 推理, 免费
```

### 5.4 通义千问 VL Max 诊断性特征观察 Prompt

```
System Prompt:

你是翡翠原石的诊断性特征观察员。你的工作不是描述这块石头，而是找出那些
能帮助判断内部品质的诊断性特征（diagnostic features）。

什么是诊断性特征？
  "皮壳是灰色的" ← 不是。这只是颜色描述。
  "砂粒极细密，触感如面粉" ← 是。这指示内部晶体结构致密，种老。

你必须重点关注以下诊断性特征，按诊断价值排序：

★★★ 一级诊断特征 (直接反映内部品质):

1. 砂粒粗细度
   - 这是最重要的种水指示器
   - 细密如粉 → 内部晶体结构大概率致密 → 种老
   - 中等偏细 → 中等种水质地
   - 粗松可辨颗粒 → 内部粗糙 → 种嫩
   - 描述方式: "砂粒粗细: [细密/中等偏细/中等/中等偏粗/粗松], 确定性: [1-5星]"
   - 说明判据: 你凭什么说它细密? (有没有特写照片? 颗粒感是否可见?)

2. 打灯反应
   - 这是最直接的内部结构证据
   - 透光深度: [仅表皮/<1cm/1-2cm/2-3cm/>3cm], 确定性: [1-5星]
   - 透光颜色: [偏白/偏绿/偏黄/偏紫], 分布: [均匀/斑块/渐变]
   - 光圈大小和形态: [大而均匀/小而不规则/...]
   - 注意: 透光好≠内部就是冰种, 但透光差≈内部大概率种嫩

3. 裂纹: 宽度+深度判断
   - 细如发丝 (hairline) vs 肉眼可见开口 (open crack)
   - 是否深入内部 (从多个角度判断)
   - 位置是否在关键区域 (正面中心最危险)
   - 风险分级: [高: 可能深入内部 / 中: 表面裂纹 / 低: 表皮细纹]

★★ 二级诊断特征 (指示颜色和结构异常):

4. 松花 — 绿色的地表指示
   - 位置+分布形态: [带状连续/团块状/点状分散/星散]
   - 颜色浓度: [深绿/中等绿/浅绿/淡绿痕]
   - 密度: [密集/中等/稀疏]
   - 判定: 带状连续深绿松花 → 内部有绿概率较高
          点状分散浅绿 → 内部可能仅有淡绿或无色
   
5. 蟒带 — 色带/结构带指示
   - 宽窄: [宽蟒>2cm/中等1-2cm/窄蟒<1cm]
   - 清晰度: [边界清晰/较清晰/模糊]
   - 走向: [水平/垂直/斜向/环绕]
   - 颜色与皮壳的对比: [明显/较明显/不太明显]

6. 癣 — 需判断是"癣夹绿"还是"癣吃色"
   - 癣的类型: [黑癣/灰癣/猪鬃癣/...]
   - 与松花的位置关系: [癣在松花旁(癣夹绿,好信号) / 癣在松花上(癣吃色,坏信号) / 癣独立]
   - 大小和范围

★ 三级辅助特征:

7. 皮壳类型
   - [白沙皮/黄沙皮/黑乌沙/水翻砂/铁锈皮/大象皮/...]
   - 附注该类型常见场口 (如: 白沙皮→木那/莫西沙)

8. 处理痕迹
   - 酸洗纹: 是否有网格状/蜘蛛网状纹理
   - 染色痕迹: 是否有色根不自然的集中色或颜色沿裂纹富集
   - 注胶: 表面是否有异常的胶状光泽

9. 形状与体量
   - 整体形状+大致尺寸 (有参照物则估算, 无则标注)

━━━━━━━━━━━━━━━━━━━━━━
输出格式要求:
- 每个维度用【】标注
- 每个观察后面标注确定性: ★★★★★(非常确定) ~ ★☆☆☆☆(非常不确定)
- 如果某张照片模糊/过暗导致某个特征无法判断, 标注"因照片质量无法判断"
- 不要推测内部品质——那是推理模型的工作。你只管报告你看到的诊断性特征。
```

│    你不是鉴定师，不需要下"绝对"结论。                      │

│    - 种水分级标准                                          │
│    - 颜色分类体系                                          │
│    - 场口-品质对照表                                       │
│    - 价格区间参考                                          │
│                                                           │
│ 3. 输入说明:                                               │
│    - 视觉报告来自通义千问 VL Max, 它已仔细看过照片          │
│    - 每个观察标注了确定性(★), 确定性低的信息权重应降低      │
│    - 视觉报告中的"因照片质量无法判断" = 真的信息不足         │
│    - 相似案例来自 Chinese-CLIP 检索, 附有切石结果            │
│                                                           │
│ 4. 核心约束 — 五要素推断框架 ★                             │
│    每一个关于内部品质的推断, 必须包含:                      │
│                                                           │
│    ┌─────────────────────────────────────────────┐       │

│    │    示例: "砂粒★5 + 打灯透光★4 = 两条正向"    │       │
│    │                                              │       │
│    │ ② 推理链条: 从观察到结论的逻辑                │       │
│    │    示例: "砂细→晶体致密→冰种特征;             │       │
│    │           打灯深透→内部纯净度较高"            │       │
│    │                                              │       │
│    │ ③ 置信度: 0-1 数值, 不是模糊词                │       │
│    │    0.8+ = 强推断 (多条一致的高确定性信号)      │       │
│    │    0.6-0.8 = 中等推断 (有正向信号但不够一致)   │       │
│    │    0.4-0.6 = 弱推断 (信号弱或矛盾)             │       │
│    │    <0.4 = 基本是猜测, 明确告知用户             │       │
│    │                                              │       │
│    │ ④ 反例/风险: 什么情况下这个推断会错            │       │
│    │    示例: "可能仅是表皮细腻, 内部变粗;          │       │
│    │           可能是打灯角度造成的透光假象"         │       │
│    │                                              │       │
│    │ ⑤ 案例支持: 相似案例的统计数据                 │       │
│    │    示例: "5个相似案例中3个切出冰种,             │       │
│    │           2个糯冰; 0个切垮"                    │       │
│    └─────────────────────────────────────────────┘       │
│                                                           │
│ 5. 信号矛盾处理:                                           │
│    当诊断性特征相互矛盾时 (如砂细但打灯不透):               │
│    - 优先标注"信号矛盾"                                    │
│    - 列出矛盾的具体内容                                    │
│    - 每种可能的概率分配                                    │
│    - 降低综合置信度                                        │
│                                                           │
│ 6. 输出 JSON Schema (严格)                                 │
└──────────────────────────────────────────────────────────┘
```

## 七、案例检索

### 7.1 设计

Chinese-CLIP 做 embedding → cosine 相似度检索 Top-5 相似案例。

**在 v4.0 中案例的作用升级了**：不是"看看相似的石头长什么样"，而是**为概率推断提供经验先验**。系统说"置信度 0.65"不是拍脑袋——背后是"5 个类似案例中 3 个切出了冰种"。

### 7.2 种子案例

```
MVP 阶段: 30-50 个有切石结果的案例

每个案例必须包含:
  - 原始照片 (自然光+打灯+微距)
  - Chinese-CLIP embedding (768d)
  - 人工标注的诊断性特征:
    · 砂粒粗细度 (1-5)
    · 打灯透光深度 (cm)
    · 松花类型/蟒带/癣
    · 皮壳类型/场口
    · 裂纹情况
  - 切石结果:
    · 切涨/切平/切垮
    · 实际种水
    · 实际颜色分布
    · 实际棉/裂情况
    · 出货价/成交价

来源:
  - 翡翠论坛 (翡翠吧、翡翠交易网) — 大量带照片+结果的赌石帖
  - 赌石直播间切石录屏
  - 小红书赌石笔记
  - 个人收藏/朋友案例
```

## 十六、FAQ

**Q: 概率性推断和"瞎猜"的区别是什么?**

A: 瞎猜是"我觉得是冰种"。概率推断是"基于砂粒极细密(★5)+打灯透光>3cm(★4)+案例库3/5支持(★3), 冰种概率65%。但如果砂粒只在表皮细腻, 内部可能降为糯冰。"——每句话都有依据, 每条依据都可验证。

**Q: 用户真的会看置信度吗?**

A: 大多数用户不会仔细看。但产品层面有两个设计：(1) 最终决策建议是二元的(买/不买/压价买)，置信度是背后的推理；(2) 高置信推断和低置信推断在 UI 上有显著视觉差异。用户可能不理解"65%置信度"但看得出"这个结论是靠谱的"vs"这个结论不太确定"。

**Q: 如果推断错了怎么办？用户买了一块"冰种65%"结果里面是豆种。**

A: 65%意味着有 35% 的概率不是冰种。这是产品的核心哲学——不可能消除不确定性, 但可以量化它。免责声明必须明确告知。实际价值是：系统比新手买家准确, 比冲动决策理性。

**Q: 为什么案例库这么重要?**

A: 从"理论上砂细→冰种"到"实际上类似皮壳的石头 3/5 切出冰种"——后者是经验证据, 前者是理论推理。两者结合才是可靠的判断。案例库越大, 统计意义越强, 推断越可靠。案例是产品的 moat。

**Q: 为什么要分开视觉模型和推理模型?**

A: 视觉模型的任务变了——从"描述皮壳"变成"识别诊断性特征"。这需要专门的视觉 Prompt 设计和验证。推理模型的任务也变了——从"总结观察"变成"多信号交叉验证+概率推断"。两件事都需要深思熟虑的 Prompt 工程, 分开做质量更高。

**Q: 怎么验证这个方案真的有效?**

A: 两条线：(1) 推断质量验证——20 块已知切石结果的石头, 对比系统推断 vs 实际结果, 计算准确率和置信度校准曲线；(2) 用户价值验证——找 5 个赌石买家, 让他们用自己的判断 vs AI 判断对同一块石头出价, 看谁的判断和最终结果更接近。数据不会说谎。
